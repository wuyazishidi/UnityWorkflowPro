# RPG 竖切片（MVP）作战计划

> 目标：用本工程的自治流水线（spec→实现→DoD 门禁→审核包）+ CC0 资源 + 轻量真实后端，
> 产出一个**能跑、能打、能存档**的可玩 RPG 竖切片，并作为持续扩成完整 RPG 的底座。
> 本文件是计划与真理来源；执行时每个功能各自落一份 `specs/NNN-*.md`。

## 0. 范围（Scope）

**In（竖切片必须有）**：玩家移动+相机跟随；一张可探索地图；1 种敌人+基础近战战斗；HP/受伤/死亡;
拾取+简单背包；本地存档读档；轻量后端（云存档 + 排行榜 API）；最小 HUD（血条/提示）。

**Out（本期明确不做，列入后续）**：剧情/任务系统、对话、多地图/传送、技能树、装备系统、
联机/同步、账号体系、音乐配乐、精致美术与动画、平衡性打磨。

## 1. 待确认的关键决策（影响速度与可行性，建议采用"推荐"项）

| # | 决策 | 推荐 | 理由 |
|---|------|------|------|
| D1 | 2D vs 3D | **2D 俯视角(top-down)** | 一晚出可玩 MVP，2D 比 3D 快数倍：无需建模/绑骨/动画状态机；Kenney 有现成 CC0 2D RPG 包 |
| D2 | 后端技术栈 | **ASP.NET Core Minimal API + SQLite** | 与 Unity 同为 C#，团队一致；SQLite 零部署、本地可跑；存档/排行榜 API 简单 |
| D3 | 输入系统 | **沿用经典 Input Manager** | 与 spec 001 一致，避免引入 New Input System 的配置摩擦 |
| D4 | 运行方式 | **有人值守长时自动跑 + 里程碑检查点** | 无人值守错误会累积；Unity 场景搭建/资源导入需要编辑器在场 |

> 美术已定：**自动找 CC0**（主源 [Kenney.nl](https://kenney.nl) —— 全 CC0、质量稳定；备选 Quaternius/OpenGameArt 仅取 CC0）。下载前核对许可证，记录到 `Assets/ThirdParty/CREDITS.md`。

## 2. 技术架构（对接现有工程）

- **客户端**：Unity 2022.3 + 现有 `Game` 程序集；逻辑用 VContainer 注入、异步用 UniTask（沿用 spec 002 范式）。
- **数据/逻辑分层**：战斗、背包、存档等核心逻辑做成**纯类/可注入服务** → EditMode 可测；MonoBehaviour 仅作表现与组合根。
- **后端**：独立目录 `server/`（ASP.NET Core Minimal API），端点 `POST /save`、`GET /save/{id}`、`GET /leaderboard`、`POST /score`；SQLite 文件库；客户端用 UniTask + UnityWebRequest 调用。
- **验证**：EditMode 测可测逻辑；新增 **PlayMode 测试 + YIUIMCP 截图能力**（见 M0），让行为可自验、并为你的 Unity 审核产出截图证据。

## 4. 自治执行方式

- 每个 spec 走 **skill `unity-feature-workflow`**：澄清→spec→plan→tasks→实现→`scripts/dod.ps1` 门禁循环→审核包。
- **硬闸门**保证：编译/测试不过不收尾、自动修。
- **检查点**：每个里程碑结束产出审核包（含**截图**）给你扫一眼；你点头再进下一个。
- 后端用独立终端运行与测试，不依赖 Unity。
- 可选：接 `anthropics/claude-code-action` 做云端 CI 兜底（需 git 已就绪，已具备）。

## 5. 风险登记（Risk Register）

| 风险 | 影响 | 缓解 |
|------|------|------|
| **"一晚出完整 RPG"预期差** | 失望 | 已对齐为"竖切片 MVP"；完整 RPG 是后续多轮 |
| Unity 场景/预制体搭建难纯代码完成 | 阻塞 | M0 加截图自验；必要处停在检查点请你在编辑器点一下 |
| CC0 资源版权/质量/下载不确定 | 阻塞/合规 | 只取 Kenney 等明确 CC0；记 CREDITS；找不到就退占位图元 |
| 无人值守错误累积 | 产出垃圾 | 采用有人值守长时跑 + 里程碑检查点(D4) |
| 后端联调/网络(本机代理) | 阻塞 | 后端本地跑；客户端请求加 `proxy` 绕过(已知代理坑) |
| 自治循环卡死/超时 | 浪费 | 硬闸门 Unity 不可用时优雅放行；单功能超时则停下报告 |
| 美术/动画使 3D 远慢于预期 | 进度 | 选 2D(D1) |

**怎么玩（开发者审核）**：在 Unity 打开本工程 → 按 **Play**（场景由 `GameRoot` 运行时自动搭建，无需布线）。
- WASD 移动（被墙阻挡），空格攻击红色敌人（击杀 +10 分）。
- 走过黄色金币自动拾取（HUD 物品数 +1）。
- 被敌人贴身掉血，HP 归零变灰、约 2 秒后中心满血重生。
- F5 存档 / F9 读档（位置/HP/背包）。
- 起后端：`dotnet run --project server/RpgServer`，游戏内 F7 交分 / F8 看排行榜。
- 左上角 HUD 显示 HP/物品/得分/操作提示。

**仍是占位美术**（彩色方块），后续可换 CC0 资源（Kenney）。这是竖切片 MVP，非完整成品 RPG。
